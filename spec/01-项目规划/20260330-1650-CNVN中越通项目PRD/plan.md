# CNVN 中越通 — 产品需求文档（PRD）

## 1. 概述

### 1.1 背景

中越两国经贸关系持续深化，越南正经历类似中国 1990 年代的高速发展期。大量越南人因求职、留学、跨境贸易的需要，有强烈的学习中文需求。然而，越南现有的中文学习市场呈现以下结构性缺口：

- 现有平台均为**职业教师主导**的机构模式，缺乏兼职/自由市场模式
- 偏远地区学生**交通成本高**（越南人均汽车拥有率约 3%，高速限速 80km/h），线下学习门槛高
- 来华留学的越南学生、在越中文导游等**兼职师资资源**完全未被激活
- 嫁入中国的越南媳妇、在华越南工作者等**特殊群体**找不到用越南语教中文的老师

CNVN（中越通）旨在填补这一市场空白，打造越南版「Preply」——**面向越南市场的中文学习双边撮合平台**。

> [!note] 项目命名
> 当前项目代号 CNVN（China–Vietnam），中文名「中越通」，后续可根据越南市场调研结果调整为越南语品牌名。

### 1.2 目标

1. 搭建连接「想学中文的越南人」与「能教中文的兼职老师」的双边市场平台
2. 以 **Web 端 MVP** 快速验证越南市场的需求和商业可行性
3. 建立**老师自主定价、平台抽成**的轻资产盈利模式
4. 通过优质的平台工具增强用户黏性，降低私下交易流失率

### 1.3 范围

**MVP 包含**：
- 老师注册、资质展示、定价设置
- 学生注册、搜索/筛选老师、预约课程
- 平台内在线视频上课（集成第三方视频通话）
- 平台支付与抽成结算
- 课后评分与评价系统
- 老师排名机制（基于学生评分）

**MVP 不包含**：
- AI 辅助教学功能（后续版本）
- 短视频内容创作/分发功能
- 移动端 App（Web 跑通后推进）
- 留学中介/升学规划功能（二期扩展）
- 国内市场（聚焦越南，不做跨国）

## 2. 用户与需求分析

### 2.1 目标用户

#### 学生端（需求方）

| 用户群体 | 学中文动机 | 特征 |
|---------|-----------|------|
| 越南在职人员 | 跨境贸易、与中资企业合作 | 需要实用口语，时间碎片化 |
| 越南青年/学生 | 赴华留学、中文考试（HSK） | 系统学习需求，价格敏感 |
| 嫁入中国的越南媳妇 | 日常沟通、融入家庭 | 需要母语（越南语）授课 |
| 越南偏远地区学习者 | 就业、出路 | 无法线下上课，强烈依赖线上 |

#### 老师端（供给方）

| 老师类型 | 来源 | 特征 |
|---------|------|------|
| 在华越南留学生 | 中国各大高校 | 双语能力强，定价低，兼职需求强 |
| 越南本地中文导游 | 越南旅游业 | 实战中文能力，熟悉本地文化 |
| 在越中国人 | 华商、外派人员 | 母语为中文，可提供沉浸式教学 |
| 越南专业中文老师 | 学校/培训机构 | 教学经验丰富，定价相对较高 |

### 2.2 核心用户故事

**US-001（学生）**：作为一名在胡志明市工厂工作的越南人，我希望能在下班后约到一位便宜的老师用越南语教我中文，不需要去市区，在手机/电脑上就能上课。

**US-002（老师）**：作为在武汉留学的越南学生，我希望利用晚上空闲时间教越南人中文赚点外快，自己定价，不被平台剥削太多。

**US-003（学生）**：作为一名想考 HSK3 的越南大学生，我希望能根据老师的评分、价格、可用时间快速筛选到合适的老师，课后还能看到学习记录。

### 2.3 功能需求

#### FR-001：老师注册与档案

**描述**：老师可以创建个人教学档案，包括自我介绍、教学风格、资质证明、课程类型和定价。

**业务规则**：
1. 老师须通过手机号或邮箱注册（支持越南/中国号码）
2. 老师自主设置小时单价（越南盾 VND 或人民币 CNY）
3. 档案包含：头像、自我介绍（越南语/中文）、擅长方向（口语/HSK备考/商务中文）、可用时间段
4. 平台不强制认证资质，但认证标识提升排名权重

#### FR-002：学生搜索与筛选

**描述**：学生可通过多维度条件筛选老师，进行比较后预约。

**业务规则**：
1. 筛选维度：价格区间、评分、擅长方向、可用时间、老师类型
2. 老师列表按「综合排名」默认排序（综合评分 + 完课数 + 响应率）
3. 支持查看老师详情页（档案 + 评价列表 + 可用日历）
4. 支持发送「试课请求」（首次接触）

#### FR-003：预约与排课

**描述**：学生选定老师后可预约具体课时，双方确认后锁定日历。

**业务规则**：
1. 老师在档案中设置可用时间段（周期性或自定义）
2. 学生选择时间 → 老师确认 → 生成课程订单
3. 支持单次课和课时包（买 N 节享折扣）
4. 课程取消须提前 24 小时，否则不退款

#### FR-004：在线视频上课

**描述**：平台内集成视频通话功能，师生在平台内完成上课。

**业务规则**：
1. 集成第三方视频 SDK（优先 Agora/声网，支持东南亚节点）
2. 视频通话界面内提供：文字聊天、文件分享（课件）、计时器
3. 课程时长自动记录，用于结算依据
4. 老师可上传课件（PDF/图片），学生课后可下载

> [!tip] 关于私下交易问题
> 通过提供便捷的平台内工具（课件上传、学习记录、计时器、评价体系），让师生感受到「平台上课更方便」，从而自然留存，而非强制约束。

#### FR-005：支付与抽成

**描述**：所有课程费用必须通过平台支付，平台从中抽成。

**业务规则**：
1. 支持支付方式：越南本地支付（MoMo、ZaloPay、银行转账）
2. 平台抽成规则（初始方案）：
   - 月完课 ≤ 20 小时：抽成 20%
   - 月完课 21–50 小时：抽成 15%
   - 月完课 > 50 小时：抽成 10%
3. 课程完成后 T+1 结算给老师
4. 学生预付款，课程完成后才向老师释放

#### FR-006：评价与排名

**描述**：学生课后对老师进行评分，平台根据综合评分动态调整老师排名。

**业务规则**：
1. 评分维度：教学质量（1–5星）、准时度、沟通友好度
2. 综合排名 = 评分 × 权重 + 完课数 × 权重 + 响应率 × 权重（具体权重待定）
3. 差评老师（综合分 < 3.0）降低曝光
4. 差评老师（综合分 < 2.5）暂停接单，平台介入审核

### 2.4 非功能需求

- **语言支持**：界面语言优先越南语，兼容中文
- **性能**：页面首屏加载 ≤ 3 秒（针对越南网络环境）
- **视频质量**：720p，支持弱网自适应降质
- **支付安全**：符合越南国家银行支付法规
- **数据合规**：遵守越南个人信息保护法规

## 3. 设计方案

### 3.1 产品架构

```
┌─────────────────────────────────────────────────────┐
│                   CNVN Web 平台                      │
├───────────────────┬─────────────────────────────────┤
│    学生端          │         老师端                    │
│  - 搜索/浏览老师   │  - 教学档案管理                   │
│  - 预约/上课       │  - 日历与可用时间                  │
│  - 支付/评价       │  - 课件上传/管理                   │
│  - 学习记录        │  - 收入统计/提现                   │
└───────────────────┴─────────────────────────────────┤
│                 平台核心服务                           │
│  撮合引擎 │ 支付网关 │ 视频通话 │ 评价系统 │ 通知服务  │
└─────────────────────────────────────────────────────┘
```

### 3.2 技术选型（初步）

| 技术领域 | 选型方向                        | 理由                  |
| ---- | --------------------------- | ------------------- |
| 前端框架 | React + Next.js             | SSR 利于 SEO，越南用户首屏体验 |
| 后端框架 | Node.js / Python FastAPI    | 待定，根据团队技术栈决定        |
| 视频通话 | Agora SDK（声网）               | 东南亚节点覆盖好，SDK 成熟     |
| 支付   | MoMo / ZaloPay API          | 越南主流支付方式            |
| 数据库  | PostgreSQL                  | 关系型，支撑复杂排课和结算逻辑     |
| 部署   | 越南本地云（VNG Cloud）或 AWS 新加坡节点 | 降低延迟，合规             |

> [!important] 技术栈待团队确认
> 上述技术选型为初步方向，具体选型需根据 R&K 的实际技术栈和开发偏好最终确定。

### 3.3 核心页面

1. **首页**：产品价值介绍 + 老师推荐列表 + 注册引导
2. **老师列表页**：搜索框 + 多维筛选 + 老师卡片列表
3. **老师详情页**：档案 + 评价 + 预约日历
4. **课程管理页（学生）**：即将上课 / 历史课程 / 学习记录
5. **课程管理页（老师）**：接单日历 / 收入统计 / 待评价
6. **视频上课页**：全屏视频 + 侧边聊天/课件 + 计时器
7. **个人中心**：基本信息 / 支付/提现 / 设置

## 4. 执行模式

### 执行模式选择

**推荐模式**：单 Agent

**选择理由**：
- 当前为 PRD 阶段文档撰写，无需多 Agent 并行
- MVP 开发由 R&K 单人主导，后续开发 Spec 同样适合单 Agent 逐步实现
- 项目初期规模小，保持流程简单高效

## 5. 实现路径（MVP 阶段）

### 阶段一：市场验证（Week 1–2）
- [ ] 完成 PRD 确认
- [ ] 制作 Landing Page（介绍产品价值，收集早期感兴趣的老师/学生）
- [ ] 对 3–5 名越南学中文用户进行访谈，验证核心痛点

### 阶段二：MVP 核心功能（Week 3–6）
- [ ] 老师注册 + 档案创建
- [ ] 学生注册 + 搜索/筛选老师
- [ ] 预约系统（老师确认）
- [ ] 集成 Agora 视频通话
- [ ] 简单支付（先用转账+手动确认，快速上线）

### 阶段三：闭环与迭代（Week 7–10）
- [ ] 课后评价系统
- [ ] 老师排名机制
- [ ] 支付网关正式集成（MoMo/ZaloPay）
- [ ] 抽成自动结算
- [ ] 邀请首批老师入驻，组织首批试课

### 阶段四：验证与扩展（Week 11+）
- [ ] 收集用户反馈，优化核心流程
- [ ] 评估移动端需求
- [ ] 研究 AI 辅助教学工具可行性

## 6. 风险与依赖

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 越南支付集成复杂，合规周期长 | 高 | 高 | MVP 先用手动转账，后续接入正式支付 |
| 冷启动：老师和学生量不足，平台无法撮合 | 高 | 中 | 优先拓展老师端（联系在华越南留学生群体），再引流学生 |
| 师生私下交易流失率高 | 中 | 中 | 提供有价值的平台工具（课件、记录、评价），增加留平台动机 |
| 越南本地法律合规（教育资质、支付牌照） | 高 | 中 | 早期以轻量信息平台定位运营，逐步完善合规 |
| 竞品跟进复刻 | 中 | 低 | 快速积累用户评价数据，形成数据护城河 |
| 视频通话质量影响用户体验 | 中 | 中 | 选择有东南亚 CDN 节点的 SDK，做弱网降级处理 |

> [!warning] 关键风险：冷启动问题
> 双边市场最难的是冷启动。建议优先从**老师端**发力：通过在华越南留学生社群、微信群等渠道招募首批 20–50 名老师，再以「低价试课」吸引学生端。

## 7. 竞品参考

| 竞品 | 模式 | 与 CNVN 差异 |
|------|------|------------|
| Preply | 全球多语言双边市场，订阅制 | 面向全球，非越南本地化，无越南语母语教中文场景 |
| italki | 全球语言学习平台 | 类似 Preply，同样缺乏越南本地化 |
| 越南现有中文 App | 职业老师机构模式 | 非兼职自由市场，价格高，无视频撮合 |

> [!tip] 差异化定位
> CNVN 的核心差异化：**越南本地化 × 自由兼职市场 × 越南语授课中文**。这三点组合在现有市场中完全空白。
