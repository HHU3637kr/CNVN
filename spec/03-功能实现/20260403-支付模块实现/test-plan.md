# 支付模块测试计划

## 1. 测试概述

### 1.1 测试目标

验证支付模块的核心功能正确性，确保：
1. 课程结算金额计算准确（价格 × (1 - 费率)）
2. 阶梯抽成费率计算正确（≤20h→20%、21-50h→15%、>50h→10%）
3. 防重复结算机制有效
4. 24h 取消规则按预期执行
5. 并发场景下数据一致性
6. 交易记录完整可追溯

### 1.2 测试范围

| 模块 | 测试内容 | 优先级 |
|------|---------|--------|
| 结算服务 | 费率计算、结算流程 | P0 |
| 钱包服务 | 入账操作、行锁 | P0 |
| 课程服务 | 结算集成、取消规则 | P0 |
| 并发安全 | 重复结算、竞态条件 | P1 |
| 边界条件 | 时区、费率边界、时间边界 | P1 |

### 1.3 测试约定

遵循项目测试约定：
- 测试数据库：`cnvn_test`（与开发库分离）
- 测试框架：`pytest-asyncio` + `httpx.AsyncClient`
- 每个测试独立 engine + session，测试后 rollback + dispose
- 测试数据使用 `uuid.uuid4().hex[:8]` 生成唯一标识
- 参考 `backend/tests/conftest.py` 中的 fixture

### 2.1 结算服务测试（settlement_service）

#### TC-SETTLE-001: 正常结算

**前置条件**：
- 教师本月无完课记录（0 小时）
- 课程状态为 `completed`，价格 500,000 VND
- 课程未结算（`settled_at IS NULL`）

## 3. API 集成测试

### 3.1 测试接口

基于 plan.md 的设计，测试以下 API：

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/v1/lessons/{id}/end` | 结束课程并触发结算 |
| POST | `/api/v1/lessons/{id}/cancel` | 取消课程（24h 规则） |
| GET | `/api/v1/wallet` | 查询钱包余额 |
| GET | `/api/v1/wallet/transactions` | 查询交易记录 |

### 3.2 测试流程

```
1. 学生注册、充值
2. 教师注册、设置可用时间
3. 学生预订课程 → 确认
4. 测试取消（>24h）→ 验证退款
5. 学生再次预订 → 确认
6. 测试取消（<24h）→ 验证不退款
7. 学生再次预订 → 确认
8. 教师结束课程 → 验证结算
9. 查询交易记录 → 验证完整性
```
