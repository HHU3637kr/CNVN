# 测试计划：课堂实时通信（WebSocket）

## 验收标准

1. **鉴权**：无效/缺失 JWT 连接 WebSocket → **连接失败或立即关闭**，无静默成功。
2. **权限**：非该课时师生连接 → **拒绝**（与 HTTP 403 语义一致）。
3. **发送与广播**：用户 A（师生）经 WS 发送聊天 → **数据库 `messages` 表有记录**；同课时在线用户 B **收到广播**（自动化或人工双人验证）。
4. **历史**：若实现 `GET .../messages`，师生可分页拉取；顺序与 `created_at` 一致。
5. **前端**：课堂页在 WS 失败时 **可见错误**；成功时可收发消息。

## 测试用例

| 用例编号 | 描述 | 输入 | 预期输出 | 边界条件 |
|---------|------|------|---------|---------|
| TC-WS-001 | 无 token 连 WS | 无 query | 连接失败 | - |
| TC-WS-002 | 错误 token | 伪造 JWT | 失败 | - |
| TC-WS-003 | 非成员 | 第三方用户 token | 拒绝 | - |
| TC-WS-004 | 师生互发 | 两客户端同 lesson | 双方收广播 + DB 有记录 | content 空串拒绝 |
| TC-WS-005 | 非法 JSON | 非 JSON 帧 | error，不落库 | - |
| TC-HTTP-001 | GET 消息 | 师生 JWT | 200，分页正确 | 若未实现 GET 则标 N/A |
| TC-HTTP-002 | 非成员 GET | 第三方 | 403 | - |

## 覆盖率要求

- **后端**：WebSocket 关键路径 + 权限分支应有自动化测试；REST（若有）沿用 pytest-asyncio。
- **功能覆盖率**：TC-WS-001～004 为 **必测**；双人广播可用 **集成测试** 或测试报告注明人工验证。

## 测试环境要求

- PostgreSQL 测试库（与 `conftest` 一致）。
- **无需** Agora 账号或额外环境变量（与本阶段设计一致）。
- 前端：本地双标签页或双浏览器（可选人工）。
